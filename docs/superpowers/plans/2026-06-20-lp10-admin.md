# LP 10 — Painel Admin (Lar & Co. Imóveis) — Plano de Implementação

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development ou superpowers:executing-plans para implementar tarefa-a-tarefa. Passos usam checkbox (`- [ ]`).

**Goal:** Adicionar um painel admin client-side à LP 10 para CRUD de imóveis + fotos, com persistência em `localStorage` e exportação do `imoveis.js`, mantendo o site no GitHub Pages.

**Architecture:** `store.js` (fonte única da verdade) mescla o seed versionado (`data/imoveis.js`) com um overlay no `localStorage`. As páginas públicas leem do `Store`; o `admin.html` faz login estético, CRUD, upload de fotos com compressão e exporta o arquivo final.

**Tech Stack:** HTML/CSS/JS puro (sem build, sem deps). Testes da lógica pura via `node:test` (Node v22). Validação de UI via Playwright servindo por `python -m http.server`.

## Global Constraints

- Sem backend, sem build, sem novas dependências; compatível com GitHub Pages.
- Diretório de trabalho: `projetos/10-imobiliaria/`.
- Seed `data/imoveis.js` é **imutável em runtime**; alterações vão para o overlay.
- Chave do overlay: `larco_admin_overlay_v1`. Sessão admin: `sessionStorage` `larco_admin_session`. Login: `admin` / `admin`.
- Fotos: compressão canvas máx. 1280px / JPEG ~0.8 → data URL.
- Identidade visual: esmeralda/creme, Fraunces + Figtree.
- Schema do imóvel inalterado (ver spec §4).
